Technical Bug Report: if_atlantic Instability and PHY Lockup on FreeBSD 15.0 (Lenovo P620)
1. Hardware Information

    Host: Lenovo ThinkStation P620

    NIC: Aquantia AQC107 10GbE (Onboard)

    PCI Identifiers: * vendor=0x1d6a (Aquantia)

        device=0x07b1 (Specific to Lenovo P620)

        subvendor=0x17aa (Lenovo)

        subdevice=0x1046

2. Environment

    OS: FreeBSD 15.0-CURRENT / 15.0-RELEASE (amd64)

    Driver Source: aqtion-freebsd (GitHub) with manual patches for FreeBSD 15 API compatibility.

3. Problem Description

The if_atlantic driver exhibits critical failures when ported to FreeBSD 15.0 on Lenovo P620 hardware. After adapting the source code to use opaque ifnet structures (if_get* functions) and addressing the pause() symbol conflict, the following issues occur:

    Massive Packet Corruption: Upon interface initialization, netstat -I reports billions of input and output errors (errs) within seconds. This suggests a major DMA misalignment or a failure in the hardware offloading logic (TSO/LRO/Checksum).

    PHY Lockup (No Carrier): Following the error storm, the NIC enters a "zombie" state. The interface reports status: no carrier despite a valid cable and link on the switch side.

    Permission Denied on Media Change: Any attempt to force media settings (e.g., ifconfig aq0 media 1000baseT) results in ifconfig: SIOCSIFMEDIA (media): Operation not permitted.

    System Degradation: The interrupt storm generated by the NIC causes significant system latency, rendering SSH sessions unresponsive and causing network stack timeouts.

4. Steps Taken to Mitigate (Unsuccessful)

    Code Adaptation: Updated aq_main.c, aq_media.c, and aq_ring.c to use if_getsoftc, if_getflags, if_getbaudrate, etc.

    Offload Disabling: Attempted to disable all hardware offloading (-tso -lro -rxcsum -txcsum -vlanhwtagging).

    Cold Boot: Performed a full power-cycle (unplugging PSU) to reset the Aquantia firmware. The NIC still fails to establish a physical link ("No Carrier") once the FreeBSD 15 driver attaches.

5. Diagnostic Output (sysctl dev.aq.0.stats)

    All MAC counters (good_octets_rcvd, good_pkts_txd) remain at 0.

    The driver attaches to the correct PCI ID, but the management interface between the driver and the NIC firmware appears broken.

    The hardware seems to protect itself by locking the PHY layer after the initial memory corruption detected by iflib.

6. Conclusion

The current iflib implementation in the atlantic driver requires deep refactoring to support the architectural changes in FreeBSD 15. Specifically, the DMA memory mapping and the initialization sequence for the AQC107 (0x07b1) appear incompatible with the new kernel network stack, leading to immediate hardware state corruption.
